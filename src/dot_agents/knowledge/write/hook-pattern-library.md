# Hook Pattern Library

> Named hook templates — curiosity gap, contrarian, before/after, listicle, story, stat, question. Reach for the pattern that matches the payload when the opening line won't come.

## Why

Hooks aren't invented from scratch each time. Working creators reuse a small set of patterns. Naming them turns "I need a good opener" into "this is a before/after — write a before/after hook." Faster, less precious, and the pattern itself does part of the work.

## The patterns

| Pattern | Shape | Example opener |
|---------|-------|----------------|
| **Curiosity gap** | Tease the payoff without revealing it | "The mistake that cost me $40k took 6 seconds to make." |
| **Contrarian** | State the consensus and reject it | "Everyone says 'find your niche.' That advice broke my writing for 2 years." |
| **Before/After** | Two timestamps, dramatic delta | "18 months ago: 200 followers. Today: 80,000. Here's the playbook." |
| **Listicle** | Numbered promise with specific count | "9 sentences I steal from copywriters." |
| **Story** | Drop into a scene mid-action | "I was 3 minutes from quitting when the email came in." |
| **Stat / data** | Hard number that contradicts a default belief | "78% of my best-performing posts were drafted in under 5 minutes." |
| **Question** | One the reader silently answers wrong | "What's the first thing you'd cut from your morning routine?" |

## How to choose

Match pattern to payload:

- Payload is a counter-intuitive lesson → **contrarian** or **stat**.
- Payload is a transformation → **before/after**.
- Payload is a list of tactics → **listicle**.
- Payload is a single insight from one moment → **story**.
- Payload is a frame the reader hasn't considered → **question**.

If two patterns fit, write both and pick by ear (or A/B test if the platform supports it).

## Anti-pattern

Stacking patterns ("Here are 7 things I learned when I lost $40k…") often dilutes. One pattern, executed cleanly, beats a hybrid. Related: [[hook-stops-the-scroll]] — one promise per hook.

## Building your own

Hook patterns are not fixed — keep a [[swipe-file-practice]] of openers that stopped *your* scroll. Reverse-engineer the pattern. Your library grows; the named taxonomy stays the same.
