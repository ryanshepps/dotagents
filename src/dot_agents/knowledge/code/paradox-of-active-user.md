# Paradox of the Active User

> Users never read manuals — they dive in, learn by doing, and only seek help when stuck.

## Key Takeaways

- Docs that users must read before acting will not be read. Teach in-flow via tooltips, empty states, and contextual hints.
- Make the product usable with zero prior reading. A first-time user clicking around should reach a useful action within seconds.
- Help content is for rescue, not introduction. Surface it when a user shows signs of being stuck, not up front.
