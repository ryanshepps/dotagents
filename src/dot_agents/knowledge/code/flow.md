# Flow

> The immersive state where users are fully engaged — design to enter, sustain, and protect it.

## Key Takeaways

- Flow needs clear goals, immediate feedback, and a challenge-skill match. Too easy = boredom; too hard = anxiety.
- Interruptions (modals, notifications, latency) break flow and cost far more than the time they directly consume — users must rebuild mental state.
- Design around uninterrupted stretches: batch notifications, defer non-critical prompts, and make undo cheap so users feel safe to move fast.
