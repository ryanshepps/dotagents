# Cognitive Bias

> Users' perceptions and decisions are systematically distorted by mental shortcuts — design around these biases, not against them.

## Key Takeaways

- Humans do not evaluate options rationally. Anchoring, framing, loss aversion, and recency all shape what users click and remember.
- Design can amplify biases ethically (e.g. highlighting defaults, showing progress) or exploit them unethically (dark patterns). Know which you're doing.
- When user behavior seems irrational, it is usually a predictable bias — name it before redesigning.
