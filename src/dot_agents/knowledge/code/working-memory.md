# Working Memory

> The short-term mental buffer users rely on mid-task — offload information to the interface so they do not have to hold it.

## Key Takeaways

- Working memory is small and fragile. Every piece of info the user must hold mentally is a liability — interruptions wipe it.
- Prefer recognition over recall. Show prior selections, keep context visible, and pre-fill known values.
- When a task needs information from step 1 at step 5, display it on step 5. Do not ask users to remember.
