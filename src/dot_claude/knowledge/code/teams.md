# Teams

How org dynamics shape software outcomes — sizing, communication overhead, coordination cost. Use when planning team shape, splitting work, or diagnosing throughput drops.

## Entries (by priority)

- [[brooks-law]] (p1) — Adding manpower to a late software project makes it later.
- [[conways-law]] (p1) — Organizations design systems that mirror their own communication structure. Team topology predicts system topology.
- [[bus-factor]] (p2) — The minimum number of team members whose loss would put the project in serious trouble.
- [[dunbars-number]] (p3) — There is a cognitive limit of about 150 stable relationships one person can maintain.
- [[linuss-law]] (p3) — Given enough eyeballs, all bugs are shallow.
- [[peter-principle]] (p3) — In a hierarchy, every employee tends to rise to their level of incompetence.
- [[prices-law]] (p3) — The square root of the total number of participants does 50% of the work.
- [[ringelmann-effect]] (p3) — Individual productivity decreases as group size increases.
